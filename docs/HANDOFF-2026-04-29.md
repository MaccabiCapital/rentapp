## Demo paths

### Listing syndication

1. Sign in → Listings → click **Set up syndication** (top-right) — or **Syndication ✓** if already enabled
2. Click **Enable syndication** → URLs appear
3. Copy the RentalSource URL (or Zillow URL) → paste into the destination portal's submission form
4. Aggregators crawl daily; `last_crawled_at` and `total_crawl_count` update per portal
5. **Do NOT submit smoke-test data to a real portal yet** — see deferred-validation section in `SPRINT-DISTRIBUTION-FEATURES.md`

### Universal accounting CSV

1. Financials → click **Accounting CSV** (next to Schedule E CSV button)
2. Downloads `rentbase-gl-YYYY-MM-DD-to-YYYY-MM-DD.csv`
3. Open in QuickBooks/Xero/Wave/FreshBooks via their generic CSV import flow
4. Account names match QBO defaults — usually no remap needed

### Public API

1. Settings → **API** tab → **Generate key** → copy the secret (shown ONCE)
2. `curl -H "Authorization: Bearer rb_live_..." https://app.rentbase.app/api/v1/properties`
3. Returns paginated JSON

## Critical rules (carry-forward)

1. units use `unit_number`; listings use `headline_rent`
2. Proof Check decision bar: all 3 buttons stay enabled regardless of risk band
3. DI engine NEVER reads name/email/phone/address
4. All FairScreen rule packs ATTORNEY-REVIEW-PENDING
5. Lease-sign tokens single-use, 14-day TTL, nulled after success
6. Syndication tokens are 32-byte base64url, public-by-design (token IS the credential, no further auth on the feed routes)
7. API keys: SHA-256 hash on lookup (deterministic; OK because input is high-entropy random); revoke flips `revoked_at`, never delete
8. **Don't submit smoke-test data to production aggregators** — wait for first paying landlord with real listing
9. **Pricing should not anchor below $50/mo** — that's the floor on the Small tier (3 doors). Going lower captures <10% of customer-perceived value
10. Sidebar IA: 8 daily items + 4 in More + Settings = 13 visible. Don't add a top-level item without removing one or folding it into a parent.
